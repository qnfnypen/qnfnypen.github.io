kubernetes安装以及简单介绍

2026年5月1日星期五 阅读时长: 3 分钟

kubernetes（K8s）是一个开源的容器编排平台，最初由Google开发，现在由CNCF（云原生计算基金会）维护。

特点

  • 自动部署、扩展和管理容器化应用
  • 提供服务发现、负载均衡和滚动更新等功能
  • 支持自动修复和自我修复机制
  • 拥有庞大的生态系统和社区支持

5. kubectl 操作集群

5.1创建 Deployment

Kubernetes Pod 是由一个或多个为了管理和联网而绑定在一起的容器构成的组。Kubernetes Deployment 检查 Pod 的健康状况，并在 Pod 中的容器终止的情况下重新启动新的容器。Deployment 是管理 Pod 创建和扩展的推荐方法。

# 使用 kubectl create 创建管理 Pod 的 Deployment
kubectl create deployment hello-node --image=registry.k8s.io/e2e-test-images/agnhost:2.53 -- /agnhost netexec --http-port=8080
# 创建一个 Deployment：
#   名称：hello-node
#   镜像：registry.k8s.io/e2e-test-images/agnhost:2.53
#   容器启动命令：/agnhost netexec --http-port=8080

# 查看 Deployment
kubectl get deployments

# 如果一直是 READY 0/1，说明 Pod 没有正常运行，可以使用以下命令查看 Pod 的详细信息
kubectl get pods -o wide
kubectl describe pod <pod-name>
kubectl logs <pod-name>

# 查看集群事件
kubectl get events --sort-by='.lastTimestamp'

# 查看 kubectl 配置
kubectl config view

5.2 创建 Service

Kubernetes Service 是一组具有相同标签的 Pod 的稳定虚拟IP地址。Service 提供负载均衡和故障转移，确保即使 Pod 发生故障，应用程序也能继续可用。默认情况下，Pod 只能通过 Kubernetes 集群内部访问，不能通过集群外部访问。要使得可以被外部访问，必须将 Pod 通过 Kubernetes Service 暴露给公网。

# 使用 kubectl expose 命令将 Pod 暴露给公网
kubectl expose deployment hello-node --type=LoadBalancer --port=8080

# 查看 Service
kubectl get services

# 对于支持负载均衡的云服务平台而言，平台将提供一个外部IP来访问该服务。在 Minikube 上，LoadBalancer 使得服务可以通过命令 `minikube service <service-name>` 来访问。
# 这将打开一个浏览器窗口，为你的应用程序提供服务并显示应用的响应
minikube service hello-node

这里的 –type=LoadBalancer 参数表明你希望将你的 Service 暴露到集群外部。测试镜像中的应用程序代码仅监听 TCP 8080 端口，因此即使使用 kubectl expose 暴露了其他端口，客户端也不能访问。

当前（Client Version: v1.36.0）支持的暴露方式：

1
2

--type='':
        Type for this service: ClusterIP, NodePort, LoadBalancer, or ExternalName. Default is 'ClusterIP'.
  1. ClusterIP: 仅在集群内部可用。这是默认类型。
  2. NodePort: 允许从集群外部访问 Service。它在每个 Node 上打开一个端口（NodePort），并将该端口的流量转发到 Service。
  3. LoadBalancer: 使用云提供商的负载均衡器将流量路由到 Service。只有在使用支持 LoadBalancer 的云平台（如 阿里云ACK、腾讯云TKE、华为云CCE、金山云KWS、AWS EKS、Azure AKS、Google GKE、火山引擎VEKS等）时才有效。
  4. ExternalName: 将 Service 映射到外部的 DNS 名称，而不是 Pod。它不代理任何流量，而是返回一个 CNAME 记录。
Type 访问范围 是否分配 ClusterIP 是否适合外部访问 常见用途
ClusterIP 集群内部 是 否 服务间调用
NodePort 节点端口 是 可以 本地测试、临时暴露
LoadBalancer 外部 LB 是 是 云上正式暴露服务
ExternalName DNS 映射 否 不是暴露服务 映射外部域名
